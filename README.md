# Avito REST API

Сервис для автоматизации создания объявлений на платформе Avito в категории "Ремонт и строительство". 
Приложение выполняет классификацию описания заказа (раздельный или комплексный ремонт) с помощью модели CatBoost 
и генерирует структурированный черновик объявления с использованием YandexGPT Lite.

## 🚀 Особенности

- **Классификация**: Предсказание типа работ на основе текста описания (CatBoost + TF-IDF).
- **Генерация контента**: Автоматическое создание продающих текстов объявлений через YandexGPT Lite.
- **FastAPI**: Современный асинхронный фреймворк с автоматической документацией (Swagger/ReDoc).
- **Модульная архитектура**: Разделение логики обработки признаков, ML-моделей и генерации текста.

## 🛠 Технологический стек

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Web Framework** | FastAPI | Построение REST API, валидация данных, авто-документация |
| **ML Model** | CatBoost 1.2.7 | Градиентный бустинг для классификации текста |
| **LLM Integration** | YandexGPT Lite (REST API) | Генерация текстов объявлений |
| **Text Processing** | Scikit-learn (TfidfVectorizer) | Векторизация и предобработка текста |
| **Data Handling** | Pandas, NumPy | Работа с табличными данными и признаками |
| **Server** | Uvicorn | ASGI-сервер для запуска приложения |

## Установка и настройка

### 1. Клонирование репозитория

```bash
git clone -b feature/Otradinskiy <repository-url>
cd avito-rest-api
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
# Для Linux/macOS:
source venv/bin/activate
# Для Windows:
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения

Создайте файл `.env` в корне проекта или экспортируйте переменные в терминале. 
Для работы модуля генерации текста **обязательно** укажите ключи доступа к Yandex Cloud:

```env
YANDEX_API_KEY=ваш_ключ_доступа
YANDEX_FOLDER_ID=ваш_folder_id
```

> **Примечание**: Без указанных переменных окружения сервис классификации будет работать, 
> но эндпоинт `/generate-draft` вернет ошибку `503`.

## Документация API

### 1. POST /generate-draft

Основной эндпоинт: принимает описание заказа, классифицирует его и генерирует черновик объявления.

**Тело запроса** (`application/json`):

```json
{
  "description": "Требуется укладка ламината в гостиной, 20 кв.м., материал заказчика"
}
```

**Успешный ответ** (`200 OK`):

```json
{
  "should_split": true,
  "probability": 0.92,
  "draft": "Заголовок: Профессиональная укладка ламината...\nОписание: ...\nПреимущества: ..."
}
```

**Поля ответа**:
| Поле | Тип | Описание |
|------|-----|----------|
| `should_split` | boolean | `true` — отдельные виды работ, `false` — комплексный ремонт |
| `probability` | float | Вероятность принадлежности к классу (0.0–1.0) |
| `draft` | string | Сгенерированный текст объявления (форматированный) |

**Коды состояний**:
| Код | Описание |
|-----|----------|
| `200` | Успешная обработка |
| `422` | Ошибка валидации входных данных |
| `503` | Сервис недоступен (модель не загружена или нет API-ключей) |
| `500` | Внутренняя ошибка сервера или ошибка генерации |
